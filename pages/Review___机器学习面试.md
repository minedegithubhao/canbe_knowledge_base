- **怎么刷**
	- 第一轮刷总流程：业务问题、数据、特征、训练、评估、优化、上线。
	- 第二轮刷任务区分：有监督/无监督、回归/分类。
	- 第三轮刷评估和泛化：准确率、召回率、F1、交叉验证、过拟合。
	- 第四轮刷经典算法：线性回归、逻辑回归、决策树、KNN、KMeans、特征缩放和梯度下降。
	- 第五轮刷强模型：随机森林、Bagging、Boosting、GBDT、XGBoost、SVM。
	- 第六轮刷工程边界：调参、网格搜索、贝叶斯优化、早停、数据泄漏。
	- 第七轮刷数据和特征：清洗、缺失值、异常值、类别编码、特征选择、防泄漏。
	- 第八轮刷评估指标细节：混淆矩阵、准确率、精确率、召回率、F1、AUC、MAE、MSE、RMSE。
	- 第九轮刷工程落地：数据集划分、baseline、误差分析、上线、监控、A/B 测试和灰度发布。
	- 第十轮刷训练机制：损失函数、交叉熵、正则化、学习曲线、SGD、Momentum、Adam。
	- 第十一轮刷无监督学习细分：KMeans、DBSCAN、聚类评估、降维/PCA 和异常检测。
	- 第十二轮刷深度学习前置：神经网络训练、反向传播、激活函数、梯度消失/爆炸、BatchNorm、Dropout。
	- 每道题都要能说出业务例子，不只背定义。

- **高频必会**
	- 机器学习项目完整流程是什么？ #card
		- 先把业务问题定义成回归、分类、聚类或排序任务，再做数据清洗、特征工程、数据集划分、模型训练、模型评估和优化，最后上线监控。关键是评估指标必须和业务目标一致。
	- 有监督学习和无监督学习有什么区别？ #card
		- 有监督学习有标签，目标是预测标签，常见任务是分类和回归；无监督学习没有标签，目标是发现数据结构，常见任务是聚类、降维和异常检测。
	- 回归和分类有什么区别？ #card
		- 回归预测连续数值，例如房价和销量；分类预测离散类别，例如是否流失和是否违约。两者评估指标不同，回归看误差大小，分类看类别判断质量。
	- 为什么不能只看准确率？ #card
		- 类别不平衡时准确率会骗人。比如欺诈检测中 99% 是正常样本，全预测正常也有 99% 准确率，但一个欺诈都抓不到，所以要看召回率、精确率、F1 和混淆矩阵。
	- 精确率和召回率怎么权衡？ #card
		- 精确率关注预测为正的样本有多少是真的正，召回率关注真实正例有多少被找出来。项目里按业务代价选主指标：怕漏报就提高召回，怕误报就提高精确率，同时用阈值和业务容量做约束。
	- F1 和 AUC 有什么区别？ #card
		- F1 是具体阈值下精确率和召回率的折中；AUC 衡量不同阈值下的整体区分能力。AUC 高不代表某个上线阈值一定好。
	- MAE、MSE、RMSE 有什么区别？ #card
		- MAE 直观且对异常值更稳，MSE 强惩罚大误差，RMSE 保留大误差惩罚且单位和目标值一致。选择哪个要看业务损失。

- **训练和评估**
	- 训练集、验证集、测试集分别做什么？ #card
		- 训练集学习参数，验证集选模型和调超参，测试集做最终隔离评估。测试集不能参与调参，否则会污染最终评估。
	- 损失函数和评估指标有什么区别？ #card
		- 损失函数是训练时优化的目标，用来指导参数更新；评估指标是判断模型效果和业务价值的标准，不一定可导。损失下降不代表业务指标一定提升。
	- 交叉熵为什么适合分类？ #card
		- 分类模型输出类别概率，交叉熵惩罚真实类别概率太低的情况。模型越确信正确类别，损失越低；自信地预测错，损失会很大。
	- L1 和 L2 正则化有什么区别？ #card
		- L1 惩罚参数绝对值之和，容易产生稀疏模型，可做特征选择；L2 惩罚参数平方和，让参数整体变小、更平滑，更常用于稳定模型。
	- 怎么用学习曲线判断过拟合和欠拟合？ #card
		- 训练集好但验证集差通常是过拟合；训练集和验证集都差通常是欠拟合。过拟合要降复杂度、加正则或加数据，欠拟合要加有效特征或提升模型能力。
	- SGD、Momentum、Adam 有什么区别？ #card
		- SGD 简单但可能震荡；Momentum 加入历史梯度方向，减少震荡并加速；Adam 结合一阶和二阶动量，自适应调整不同参数的学习率，常作为神经网络默认起点。
	- 神经网络的训练过程是什么？ #card
		- 先前向传播得到预测，用损失函数计算误差，再通过反向传播计算每个参数的梯度，最后用优化器更新参数。训练中还要用验证集监控泛化，避免只把训练集拟合好。
	- 反向传播的核心思想是什么？ #card
		- 反向传播先从损失函数出发，再用链式法则从输出层向输入层逐层计算梯度，让每个参数知道自己对损失的影响，优化器再据此更新参数。
	- 为什么会出现梯度消失和梯度爆炸？ #card
		- 深层网络反向传播时，梯度会按链式法则连续相乘。很多小梯度相乘会越来越小，导致前面层学不动；很多大梯度相乘会越来越大，导致训练震荡或 NaN。
	- BatchNorm 和 Dropout 有什么作用？ #card
		- BatchNorm 通过标准化中间激活稳定训练、加快收敛；Dropout 训练时随机丢弃部分神经元，减少共适应，主要用于缓解过拟合。
	- 为什么要做交叉验证？ #card
		- 单次划分可能偶然偏好某个模型。交叉验证通过多次划分和评估取平均，让结果更稳定，也更适合模型选择和超参调优。
	- 过拟合和欠拟合怎么判断？ #card
		- 训练集好但验证集差是过拟合，训练集和验证集都差是欠拟合。过拟合通常用更多数据、正则化和降复杂度处理；欠拟合通常增加特征或模型复杂度。

- **经典算法**
	- 线性回归和逻辑回归有什么区别？ #card
		- 线性回归做连续值预测，逻辑回归做分类概率预测。两者都是线性模型，但输出含义、损失函数和评估指标不同。
	- 决策树为什么容易过拟合？ #card
		- 决策树可以不断切分训练数据，容易把噪声和偶然样本也记住。常见处理是限深、限制叶子样本数、剪枝、交叉验证或使用集成树。
	- KNN 为什么需要特征缩放？ #card
		- KNN 依赖距离计算，数值范围大的特征会主导距离，所以要归一化或标准化，并且只能在训练集上拟合缩放参数。
	- KMeans 的 K 怎么选？ #card
		- 可以结合肘部法、轮廓系数、业务先验和抽样检查。项目里还要看簇是否稳定、可解释、可被业务使用。
	- 归一化和标准化有什么区别？ #card
		- 归一化通常缩放到固定区间，标准化转换为均值为 0、标准差为 1。两者都用于减少特征尺度干扰，但不能用全量数据拟合参数。
	- 梯度下降为什么需要学习率？ #card
		- 学习率控制每次参数更新步长，太大可能震荡或发散，太小会收敛很慢，需要结合损失曲线和验证集效果调整。
	- ID3、C4.5、CART 有什么区别？ #card
		- ID3 用信息增益，C4.5 用信息增益率，CART 分类树用基尼指数，CART 回归树用平方误差或方差减少。
	- 决策树如何选择划分特征？ #card
		- 目标是让子节点更纯或误差更低。分类树看信息增益、信息增益率或基尼指数，回归树看平方误差或方差减少。
	- 欧氏距离、曼哈顿距离、闵可夫斯基距离有什么区别？ #card
		- 欧氏距离看直线距离，曼哈顿距离看各维绝对差之和，闵可夫斯基距离是统一形式，参数 p 为 1 时是曼哈顿，p 为 2 时是欧氏。
	- 随机森林为什么能缓解决策树过拟合？ #card
		- 它通过样本随机和特征随机训练多棵不同的树，再投票或平均，降低单棵树对噪声的敏感性，从而降低方差。
	- Bagging 和 Boosting 有什么区别？ #card
		- Bagging 并行训练模型，重点降方差，代表是随机森林；Boosting 串行训练模型，后一个修正前一个错误，重点降偏差，代表是 GBDT 和 XGBoost。
	- GBDT 和 XGBoost 有什么区别？ #card
		- XGBoost 是 GBDT 的增强实现，加入正则化，使用二阶梯度，并支持列采样、缺失值默认方向和并行优化。
	- SVM 的核心思想是什么？ #card
		- SVM 寻找最大间隔分类边界，真正决定边界的是支持向量；软间隔处理噪声，核函数处理非线性。
	- 模型调参的完整流程是什么？ #card
		- 先固定数据划分和评估指标，建立 baseline，确定搜索空间，用验证集或交叉验证搜索参数，结合误差分析确认收益，最后只在独立测试集上做一次最终评估。
	- 网格搜索和贝叶斯优化有什么区别？ #card
		- 网格搜索穷举候选组合，简单可复现但成本高；贝叶斯优化根据历史试验优先尝试有希望的区域，适合搜索空间大、训练成本高的场景。
	- 早停为什么能缓解过拟合？ #card
		- 它监控验证集指标，当训练集继续变好但验证集不再提升时停止训练，避免模型继续拟合训练噪声。
	- 如何避免调参导致测试集泄漏？ #card
		- 训练集学习参数和拟合预处理器，验证集调参和模型选择，测试集只做最终一次评估；不能用测试集选参数、早停或拟合预处理。
	- 机器学习项目中特征工程怎么做？ #card
		- 先明确业务目标和预测时点，再做数据清洗、缺失值和异常值处理、类别编码、特征构造、特征缩放和特征选择，最后检查线上可用性并防止数据泄漏。
	- 缺失值应该怎么处理？ #card
		- 先看缺失比例、字段类型和缺失原因，再选择删除、统计填充、特殊类别、缺失指示特征或模型原生处理。填充值只能从训练集学习。
	- 类别特征怎么编码？ #card
		- 无序低基数常用独热编码，有序类别用有序编码，树模型可用标签编码，高基数类别可考虑频次编码、目标编码或 embedding，并注意防泄漏和未知类别。
	- 特征选择有哪些方法？ #card
		- 常见有过滤法、包装法、嵌入法和业务审查。最终要用验证集或交叉验证确认泛化提升，并检查特征是否泄漏。
	- 特征工程如何避免数据泄漏？ #card
		- 特征只能使用预测时可获得的信息；缺失填充、缩放、编码和特征选择都只能在训练集拟合，再应用到验证集、测试集和线上。

- **无监督学习**
	- KMeans 和 DBSCAN 有什么区别？ #card
		- KMeans 需要指定 K，速度快，适合大规模和近似球形簇；DBSCAN 基于密度，不需要指定簇数，可以识别噪声和不规则簇，但对 eps、min_samples 和高维距离敏感。
	- PCA 降维的核心思想是什么？ #card
		- PCA 找到数据方差最大的几个正交方向，把高维特征投影到少数主成分上，用更少维度保留主要变化信息。它是无监督方法，不看标签。
	- 聚类效果怎么评估？ #card
		- 可以看轮廓系数、SSE 肘部法、簇稳定性、可解释性和人工抽检。无监督没有标准答案，所以数学指标必须结合业务判断。
	- 无监督异常检测怎么做？ #card
		- 先定义正常样本表示，再用距离、密度、孤立程度、重构误差或聚类噪声识别异常，最后结合业务规则、人工样本和后续标签回流验证。
	- 降维会不会丢失信息？ #card
		- 会。降维本质是压缩表示，通常牺牲部分细节换取去噪、可视化、加速或降低维度灾难。是否值得要看下游任务效果和解释性要求。

- **工程落地**
	- 机器学习项目如何从训练走到上线？ #card
		- 先固定业务目标、数据版本、特征版本和评估指标，用 baseline 跑通训练和评估，再做调参和误差分析。上线时保存模型和预处理器，封装批处理或在线服务，先灰度或 A/B 测试，再监控服务、数据、输出和业务效果。
	- 模型上线后如何监控？ #card
		- 分层监控服务稳定性、特征分布和缺失率、模型输出分布、数据漂移、业务指标，以及标签回流后的真实效果，并配套告警和回滚。
	- 离线指标好但线上效果差怎么排查？ #card
		- 先查离线评估是否泄漏，再查训练和线上特征是否一致，接着看数据漂移、模型版本、服务链路、阈值和业务策略，最后用线上错误样本做误差分析。
	- 模型效果衰减怎么办？ #card
		- 先确认衰减是否真实，排除监控口径、标签延迟和服务异常，再检查数据漂移、概念漂移和错误样本分布，必要时调整阈值、补数据、重训、灰度验证或回滚。
