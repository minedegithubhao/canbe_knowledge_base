	- ## 📝 面试题（自问自答）
		- 什么是 RAG？ #card #面试背诵汇总/大模型/RAG
			- RAG 是一种**大模型应用架构**：
				- 先从**外部知识库 / 文档 / 数据库**里**检索**相关信息
				- 把检索到的内容**拼进 Prompt**里
				- 再让大模型**基于检索内容生成回答**
		- RAG为什么出现 / 解决什么问题？#card #面试背诵汇总/大模型/RAG
			- 原生大模型有 3 个致命问题：
				- **知识过时**：训练数据截止到某个时间点，不知道新信息
				- **幻觉（Hallucination）**：瞎编答案、编造事实
				- **无法使用私有数据**：不能直接读你公司内部文档、业务数据RAG 就是**不重新训练模型**，用 “检索 + 生成” 解决这三个问题。
		- 标准 RAG 的完整流程是什么？#card #面试背诵汇总/大模型/RAG
			- **用户提问**：用户输入 query
			- **查询预处理**：query 改写、纠错、扩展
			- **混合检索（向量检索 / 关键词检索）**：从知识库召回相关文档片段
			- **结果融合**：多源检索结果合并、去重
			- **重排序（Rerank）**：用小模型精排，把最相关的放前面
			- **构建 Prompt**：把系统提示 + 检索内容 + 用户问题拼在一起
			- **大模型生成**：模型**只依据检索内容**回答，减少幻觉
			- **结果返回**：输出答案给用户
		- RAG 里常用的检索方式有哪些？什么时候用向量检索？什么时候用关键词检索？#card #面试背诵汇总/大模型/RAG
			- 常用检索方式
				- **关键词检索（如 BM25）**
				- **向量检索（语义检索）**
			- 向量检索（语义检索）
				- 适合：**口语化查询、自然对话、模糊意图、同义表述**
			- 关键词检索（精确匹配）
				- 适合：**专业术语、专业词精确匹配场景**
			- 企业级最佳实践
				- 关键词召回 + 向量召回 → 结果融合 → Rerank 重排序
		- RAG 里大模型还出现幻觉，怎么排查和解决？#card #面试背诵汇总/大模型/RAG #面试背诵汇总/重点
			- 确定病因——先看「检索召回」有没有拿到正确信息（看召回的 chunks **和用户问题是否真相关**）
			- 没有正确信息
				- 检索策略：是否对关键字词、专业术语不敏感
					- M25 + 向量混合检索
				- 切片策略：块太小语义不全？，块太大语义模糊？
					- 优化 chunk 大小与 overlap，多层级切分
				- 重排序策略：
					- 最相关的有没有靠前
					- 扩大重排范围（比如从 20 条提到 30 条）
					- 增加精排序、缩小精排召回数量（Top10减少到Top3）
			- 有正确信息
				- [[提示词工程]]
					- **只根据检索内容回答如果检索内容里没有答案，必须说 “根据现有资料无法回答”，不允许编造**，不要给模型瞎编的机会
				- 上下文注入
					- 将最相关的片段放在 [[prompt]] 的最前面
			- Query 改写
			- 引入评估体系
				- [[PAGAS]] 评估框架
