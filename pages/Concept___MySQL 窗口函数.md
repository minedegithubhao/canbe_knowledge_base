- **一句话定义**
	- MySQL 窗口函数是在不折叠明细行的前提下，对每一行关联的一组窗口数据做排名、聚合或前后行计算。

- **解决的问题**
	- 在保留原始行的同时计算排名、组内 Top N、累计值、占比、前后行差值。
	- 避免为了排名和分组统计写复杂子查询。

- **核心机制**
	- `over` 定义窗口范围。
	- `partition by` 定义分组窗口。
	- `order by` 定义组内顺序。
	- `row_number`、`rank`、`dense_rank` 用于排名。
	- `lag`、`lead` 用于访问前后行。

- **典型流程**
	- 明确是否按某个字段分组。
	- 明确组内排序字段。
	- 选择排名、聚合或前后行函数。
	- 外层再过滤排名结果，例如取每组 Top N。

- **项目中怎么用**
	- 分析用户行为、推荐候选排序、检索日志 Top N、评估任务分组排名时都很常用。
	- 面试里常用于回答“每个分组取最高分/最近一次/Top N 怎么写”。

- **面试表达**
	- 窗口函数和 `group by` 有什么区别？ #card
		- `group by` 会把多行聚合成一行，窗口函数保留每一行，同时给每一行附加基于窗口计算出来的值。

- **常见误区**
	- 误区：窗口函数就是聚合函数。
		- 窗口函数可以使用聚合函数，但关键差异是不会天然折叠明细行。
	- 误区：`rank` 和 `dense_rank` 一样。
		- `rank` 遇到并列会跳号，`dense_rank` 遇到并列不跳号。
