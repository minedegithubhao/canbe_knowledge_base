- **一句话定义**
	- Python 操作 Redis 是指用 Python 客户端读写 Redis 的字符串、哈希、列表、集合、有序集合等数据结构，用于缓存、计数、会话、排行榜和轻量队列。

- **解决的问题**
	- 为高频读取数据提供缓存，降低数据库压力。
	- 保存短生命周期状态，例如验证码、登录态、限流计数和任务状态。
	- 利用 Redis 数据结构实现计数、去重、排行榜和队列。

- **数据结构选择**
	- 字符串：缓存单个值、计数器、开关和序列化对象。
	- 哈希：保存对象的多个字段，适合用户信息、配置项和状态记录。
	- 列表：按顺序保存元素，可用于简单队列或最近记录。
	- 集合：去重集合，适合标签、去重、共同元素计算。
	- 有序集合：带分数排序，适合排行榜、延迟队列和按权重取 Top。
	- 过期时间：缓存和临时状态要设置合理 TTL，避免永久堆积。

- **工程要点**
	- 使用连接池复用连接，避免每次请求都创建连接。
	- 统一序列化格式，避免不同模块写入格式不兼容。
	- 设计 key 命名规范，包含业务、对象类型和标识。
	- 对缓存穿透、击穿和雪崩要有兜底策略。
	- 涉及多个 Redis 操作时要考虑原子性，必要时使用事务、Lua 或合适的数据结构。

- **面试表达**
	- Python 操作 Redis 常见数据结构怎么选？ #card
		- 单值用字符串，对象字段用哈希，顺序队列用列表，去重关系用集合，排行榜和带权排序用有序集合；缓存类 key 要设置 TTL。

- **常见误区**
	- 误区：Redis 只是 key-value 缓存。
		- Redis 的价值很大一部分来自数据结构和原子操作。
	- 误区：缓存只要写进去就行。
		- 还要考虑过期策略、一致性、击穿、雪崩、序列化和 key 规范。
