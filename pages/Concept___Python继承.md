- **一句话定义**
	- Python 继承是子类复用和扩展父类公共能力的机制，常和方法重写、super 调用父类逻辑、MRO 方法解析顺序一起出现。

- **解决的问题**
	- 抽取多个类的共同逻辑。
	- 在保留通用行为的同时，让子类提供差异化实现。
	- 让公共接口在多个相关对象之间保持一致。

- **核心机制**
	- 子类可以访问父类公共属性和方法。
	- 子类可以重写父类方法，改变具体行为。
	- super 用于调用父类或 MRO 链上的下一个实现。
	- Python 支持多继承，复杂多继承时方法查找顺序由 MRO 决定。
	- 双下划线私有成员会被名称改写，不适合作为子类扩展接口。

- **典型流程**
	- 先抽取父类公共行为。
	- 子类继承父类并补充差异属性。
	- 需要保留父类初始化逻辑时，用 super 调用父类构造方法。
	- 子类重写方法时，明确是完全替换还是先复用父类再增强。
	- 多继承时检查 MRO，避免调用顺序和预期不一致。

- **项目中怎么用**
	- 适合表达明确的“是一种”关系，例如不同存储客户端继承同一基类。
	- 如果只是复用少量代码，组合通常比继承更灵活。
	- 多继承容易增加理解成本，工程里应谨慎使用。

- **面试表达**
	- super 的作用是什么？ #card
		- super 用来沿着 MRO 调用父类或下一个类的方法，常见于子类构造方法中复用父类初始化逻辑，也常用于重写方法时保留父类行为。

- **常见误区**
	- 误区：继承越多复用越好。
		- 继承会引入层级耦合，过深继承反而降低可维护性。
	- 误区：多继承就是简单从左到右找方法。
		- Python 使用 MRO 决定解析顺序，复杂场景要显式检查。
