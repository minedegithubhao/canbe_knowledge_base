- **一句话定义**
	- Python 可变对象可以在原对象上改变内容，不可变对象一旦创建就不能原地改变内容。

- **解决的问题**
	- 判断函数调用是否可能产生副作用。
	- 解释默认参数、浅拷贝、哈希能力和容器嵌套的常见坑。
	- 选择适合用作 dict key 或 set 元素的对象。

- **核心机制**
	- list、dict、set 是典型可变对象。
	- int、float、bool、str、tuple 通常按不可变对象理解。
	- 可变对象修改时对象身份通常不变，不可变对象看似修改时通常会创建新对象或重新绑定变量名。
	- 可哈希对象必须在生命周期内哈希值稳定，所以可变对象通常不能作为 dict key。
	- 函数默认参数只在函数定义时创建一次，可变默认参数会在多次调用间共享状态。

- **典型流程**
	- 需要原地增删改，用可变对象。
	- 需要稳定键、不可变记录或避免副作用，用不可变对象。
	- 向函数传入可变对象时，明确函数是否会修改它。
	- 需要隔离嵌套对象时，区分浅拷贝和深拷贝。

- **项目中怎么用**
	- 配置、缓存、聚合统计常用 dict，但要避免在多个调用之间意外共享同一个 dict。
	- 数据清洗中 list 和 set 很常见，但函数边界要说明是否会原地修改传入数据。
	- 面向接口设计时，返回不可变结构可以减少外部误改内部状态。

- **面试表达**
	- Python 可变对象和不可变对象有什么区别？ #card
		- 可变对象能在原对象上修改内容，例如 list、dict、set；不可变对象创建后不能原地改变，例如 int、str、tuple。这个差异会影响函数副作用、默认参数、哈希能力和拷贝行为。

- **常见误区**
	- 误区：函数传参是简单的值传递或引用传递二选一。
		- 更准确地说，Python 传递的是对象引用的绑定；能不能影响外部对象，取决于是否修改了可变对象本身。
	- 误区：默认参数每次调用都会重新创建。
		- 默认参数在函数定义时创建一次，使用可变对象做默认值容易保留上次调用的状态。
