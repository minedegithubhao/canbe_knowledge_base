- **一句话定义**
	- Python 是一门以对象模型、动态类型和高可读性语法为核心的通用编程语言，适合脚本、数据处理、Web 后端、自动化和 AI 工程。

- **解决的问题**
	- 用较少样板代码表达业务逻辑、数据处理逻辑和工程胶水逻辑。
	- 通过丰富标准库和第三方生态快速连接文件、数据库、网络服务和机器学习框架。
	- 在面试和项目中考察对象模型、数据结构、函数机制、异常处理和工程习惯。

- **核心机制**
	- 一切皆对象，变量名本质上是对象引用。
	- 动态类型让变量不固定类型，但对象本身有明确类型。
	- 可变对象和不可变对象的差异会影响传参、副作用、哈希和容器使用。
	- 函数是一等对象，可以作为参数传递、作为返回值返回，也可以被闭包和装饰器包装。
	- 类和对象用于组织复杂状态，面向对象关注封装、继承和多态。
	- 异常机制用于表达失败路径，工程代码应明确捕获范围和资源释放方式。
	- 文件、上下文管理器、迭代器和生成器是 Python 工程代码中常见的资源管理和流式处理基础。
	- 数据库、缓存、API 调用和环境管理是 Python 从语法走向工程项目时最常见的外部连接能力。

- **典型流程**
	- 选择合适数据结构保存数据。
	- 用函数拆分可复用逻辑。
	- 用异常处理边界错误。
	- 用模块、包和依赖管理组织工程代码。
	- 用测试和日志验证行为并定位问题。

- **项目中怎么用**
	- 数据处理脚本中，常用 list、dict、set 做清洗、映射、去重和聚合。
	- Web 或任务系统中，常用装饰器做鉴权、日志、缓存、重试和事务边界。
	- 复杂业务对象、客户端和处理器常用类封装状态和行为。
	- 工程代码中，异常处理和资源释放决定程序在失败场景下是否稳定。
	- 大文件和批量数据处理常用迭代器或生成器降低内存压力。
	- 连接 MySQL、Redis 和外部 API 时，要把连接生命周期、超时、异常、凭据、日志和重试作为工程边界处理。

- **面试表达**
	- 学 Python 基础不能只背语法，要能解释对象引用、可变性、哈希表、函数参数、闭包、装饰器、面向对象、上下文管理器、迭代器和异常处理背后的机制。
	- Python 面试常见追问不是“这个方法怎么写”，而是“为什么这样设计、什么时候会踩坑、工程里怎么规避”。

- **常见误区**
	- 误区：Python 变量就是装值的盒子。
		- 更准确地说，变量名绑定到对象；赋值通常改变绑定关系，不一定修改对象本身。
	- 误区：语法会用就等于 Python 基础扎实。
		- 面试更看重对象模型、作用域、可变性、异常边界和工程可维护性。
