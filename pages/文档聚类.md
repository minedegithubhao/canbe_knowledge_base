	- ## 🧠 一句话说清楚（费曼）
		- 把整篇文档按照**主题**分为一堆又一堆
	- ## 💘企业开发场景
		- 抽取整篇文档所有文本（不要图片） → 1个[[文本向量]]（BGE/Embedding） → [[KMeans]] 聚类  →  主题/簇 → 每个主题/簇一个 [[Collection]]
	- ## ⚠️易错/易混点（复习必看）
		- [[文本聚类]] VS **文档聚类**
			- 相同点：两者**底层技术完全一样**，都是把文本 → 文本向量 → [[聚类算法]]
			- 异同点：文本聚类关注的可能是细节（细粒度），文档聚类关注的是主题（粗粒度）
	- ## 🔁核心原理/流程（极简版）
		- ### 1. 获取整篇文档的全文文本
			- 从 PDF/Word 里**把整篇文本抽出来**
			- 不切块、不拆分、不拆图
			- 每篇文档 = 一段长文本
		- ### 2. 把文档转成向量
		  用**文本编码器**把整篇文档变成**一个[[文本向量]]**：
			- [[BERT]]
			- TextCNN
			- 最简单、企业最常用：**text-embedding-ada / bge-small / bge-base**
			  > 一篇文档 = 1 个向量
		- ### 3. [[聚类算法]]
		- ### 4. 输出聚类结果
		  每篇文档得到一个 **簇号（0、1、2、3…）**
			- 簇 0：财务类文档
			- 簇 1：人事类文档
			- 簇 2：IT 类文档
			- …
		- ### 5. 按簇建 [[Collection]]
			- 簇 0 → 财务知识库
			- 簇 1 → 人事知识库
			- 簇 2 → IT 知识库
