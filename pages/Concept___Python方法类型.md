- **一句话定义**
	- Python 方法类型主要包括实例方法、类方法和静态方法，区别在于方法默认接收什么对象，以及是否需要访问实例状态或类状态。

- **解决的问题**
	- 区分一段逻辑应该绑定到实例、类，还是只是放在类命名空间下。
	- 避免把不需要对象状态的工具逻辑写成实例方法。
	- 清晰表达方法和实例属性、类属性之间的关系。

- **核心机制**
	- 实例方法默认接收当前实例，通常第一个参数写作 self。
	- 类方法使用 classmethod 装饰，默认接收类对象，通常第一个参数写作 cls。
	- 静态方法使用 staticmethod 装饰，不自动接收实例或类。
	- 类属性属于类，实例访问类属性时，如果实例有同名属性，会优先访问实例属性。
	- 通过实例给同名属性赋值，通常只是创建或修改实例属性，不等于修改类属性。

- **典型流程**
	- 需要读写实例属性，用实例方法。
	- 需要读写类属性、构造替代构造器，或和具体子类绑定，用类方法。
	- 不需要实例和类状态，但逻辑和类强相关，用静态方法。
	- 只是通用工具函数且和类关系不强，放模块级函数更清晰。

- **项目中怎么用**
	- 实例方法承载对象行为，例如客户端请求、处理器执行。
	- 类方法常用于基于配置创建对象、统计类级别状态。
	- 静态方法适合放少量和类语义强相关的纯工具逻辑。

- **面试表达**
	- 实例方法、类方法、静态方法有什么区别？ #card
		- 实例方法接收 self，操作实例状态；类方法接收 cls，操作类状态或构造类对象；静态方法不自动接收 self 或 cls，只是放在类命名空间下的普通函数。

- **常见误区**
	- 误区：静态方法用于操作类属性。
		- 需要操作类属性时更适合类方法；静态方法不自动拿到类对象。
	- 误区：通过实例改类属性就是改了类属性。
		- 通过实例赋同名属性通常会生成实例属性，遮蔽类属性。
