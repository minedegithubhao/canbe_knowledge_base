- **一句话定义**
	- Routing 是根据输入内容、任务类型、上下文或规则，把请求分发到不同模型、工具、链路或处理策略的机制。

- **解决的问题**
	- 不同问题适合不同流程，固定链路容易低效或错误。
	- 系统需要在 RAG、工具调用、人工转接、普通问答、拒答之间选择。
	- 复杂 Agent 需要根据状态决定下一步动作。

- **实现方式**
	- 规则路由：基于关键词、字段、权限、业务规则。
	- LLM 路由：让模型判断问题类别和下一步。
	- 向量路由：用 embedding 判断语义相似度，把问题送到对应知识库或工具。
	- 分类模型路由：用专门训练的分类器做稳定决策。

- **项目中怎么用**
	- 客服 Agent 可先判断是 FAQ、订单查询、投诉、人工转接还是知识库外问题。
	- 多知识库 RAG 可根据主题路由到不同 Collection 或 MOC。
	- 图文知识库可根据问题类型选择文本检索、图片检索或混合检索。

- **面试表达**
	- Routing 在 Agent 中解决什么问题？ #card
		- 它解决“下一步走哪条链路”的问题。系统根据输入和状态选择模型、工具、知识库或人工流程，让 Agent 不必所有请求都走同一套固定链路。

- **常见误区**
	- 误区：Routing 必须由 LLM 做。
		- 简单、高频、强规则场景优先用规则或分类器，LLM 适合语义复杂、规则难写的场景。
	- 误区：路由错了后面还能修回来。
		- 路由是上游决策，错误路由会让后续检索、工具和生成都偏离。
