- **一句话定义**
	- conda 环境管理是用独立环境隔离 Python 版本和项目依赖，避免不同项目之间的包版本互相污染。

- **解决的问题**
	- 不同项目需要不同 Python 版本或包版本。
	- AI、数据处理和后端项目依赖复杂，容易出现版本冲突。
	- 项目需要在另一台机器或团队成员环境中复现。

- **核心流程**
	- 为项目创建独立环境。
	- 激活环境后安装依赖。
	- 用固定依赖文件记录关键包版本。
	- 开发、测试和运行都在同一环境中执行。
	- 项目结束或环境损坏时删除重建。

- **工程要点**
	- 不要把所有包都安装到 base 环境。
	- 每个项目尽量使用独立环境名。
	- 重要项目要记录 Python 版本和核心依赖版本。
	- 依赖冲突时优先新建干净环境验证，不在旧环境里反复覆盖。
	- 环境文件应服务复现，但不要把本地绝对路径和私有凭据写进去。

- **面试表达**
	- conda 环境管理解决什么问题？ #card
		- 它解决项目依赖隔离和环境复现问题。不同项目可以使用不同 Python 和包版本，避免全局环境污染，也便于定位依赖冲突。

- **常见误区**
	- 误区：只要 pip install 成功，环境就没问题。
		- 当前能安装成功不代表可复现，也不代表和其他包兼容。
	- 误区：base 环境方便，所有项目共用就行。
		- 长期会导致依赖冲突，难以判断哪个项目需要哪个版本。
