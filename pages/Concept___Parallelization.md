- **一句话定义**
	- Parallelization 是把可以独立执行的子任务并行运行，再汇总结果的 LLM 应用和 Agent 编排模式。

- **解决的问题**
	- 串行执行多个独立子任务会增加延迟。
	- 多个检索、分类、校验或生成分支可以同时运行，提高整体响应速度。
	- 对复杂任务可以让多个分支分别给出候选，再做聚合或裁决。

- **典型流程**
	- 将任务拆成互不依赖的子任务。
	- 并行调用多个模型、工具、检索器或 Prompt 分支。
	- 收集结果。
	- 用规则、投票、Rerank、LLM 汇总或人工审核做合并。
	- 设置超时、失败降级和成本上限。

- **项目中怎么用**
	- RAG 中可以并行跑向量检索、BM25、metadata 过滤和 query 改写候选。
	- Agent 中可以并行做资料搜索、事实核验和格式检查。
	- 评估系统中可以并行执行多个 case 或多个指标。

- **面试表达**
	- Parallelization 在 Agent 里解决什么问题？ #card
		- 它解决独立子任务串行导致的延迟问题。只要子任务之间没有强依赖，就可以并行执行，再用聚合策略汇总结果；但要控制成本、超时和错误合并风险。

- **常见误区**
	- 误区：所有步骤都能并行。
		- 有依赖关系的步骤仍要串行，例如先检索再基于证据回答。
	- 误区：并行一定更准。
		- 并行主要降低延迟或提高覆盖，最终质量取决于聚合和校验。
