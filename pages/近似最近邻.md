- 背景
	- 当数据库中有数百万、数十亿的向量时，将查询向量与每一个向量都计算一遍（称为**暴力搜索**或**精确搜索**）是极其耗时的。为了解决这个问题，**ANN (Approximate Nearest Neighbor)** 算法应运而生。
- 核心概念:
	- ANN 搜索的核心思想是**牺牲极小的精度来换取巨大的性能提升**。它不保证找到绝对最相似的邻居，但能以极快的速度（通常是毫秒级）找到一个或多个“足够相似”的邻居。
- 常见ANN算法
	- | ANN 算法 | 特点 | 适用场景 |
	  | ---- | ---- | ---- |
	  | **HNSW** | 召回率高、速度快、支持动态数据 | Milvus 默认首选，RAG 通用场景 |
	  | **IVF_FLAT** | 适合海量数据，需调参，召回率稳定 | 超大规模向量库（亿级以上） |
	  | **ANNOY** | 轻量、内存占用小，适合静态数据 | 推荐系统、离线检索 |
	  | **FLAT** | 暴力精确搜索（非 ANN） | 小数据集、需要 100% 召回 |
- 有个 [[KNN]] 两者的区别
	- **KNN**：用于**机器学习预测任务**（比如：根据用户特征分类、房价回归预测），是「模型」。
	- **ANN**：用于**向量检索任务**（比如：RAG 知识库检索、推荐系统、图像以图搜图），是「检索工具」。
