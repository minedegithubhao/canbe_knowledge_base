- **一句话定义**
	- Python 迭代器是按需返回下一个元素的对象，是 for 循环、生成器和惰性处理的基础。

- **解决的问题**
	- 统一遍历不同容器和数据流。
	- 按需取数据，避免一次性把全部数据放进内存。
	- 为自定义对象提供可被 for 循环消费的协议。

- **核心机制**
	- 可迭代对象表示可以被迭代，例如 list、tuple、dict、set、str 和文件对象。
	- 迭代器表示能不断返回下一个元素，直到没有数据。
	- for 循环会从可迭代对象拿到迭代器，再反复取下一个元素。
	- 迭代结束时，迭代器会发出停止信号，for 循环据此结束。
	- 自定义类迭代器通常需要维护当前位置状态。

- **典型流程**
	- 准备一个可迭代对象。
	- for 循环获取并消费元素。
	- 如果需要自定义遍历行为，为对象提供迭代协议。
	- 如果只是简单惰性生成序列，优先考虑 [[Concept/Python生成器]]。

- **项目中怎么用**
	- 逐行读取大文件时，文件对象可以被迭代，避免一次性读取所有内容。
	- 批量数据处理时，迭代器适合流式消费数据。
	- 自定义数据集、分页读取器、消息消费者都可以设计成可迭代对象。

- **面试表达**
	- 可迭代对象和迭代器有什么区别？ #card
		- 可迭代对象表示能被遍历；迭代器表示真正按需产出下一个元素的对象。for 循环会先从可迭代对象获得迭代器，再不断取值直到结束。

- **常见误区**
	- 误区：所有能 for 循环的对象本身都是迭代器。
		- 能 for 循环说明它可迭代，但它不一定本身就是迭代器。
	- 误区：迭代器可以无限重复从头遍历。
		- 很多迭代器被消费后不能自动回到开头，需要重新创建。
