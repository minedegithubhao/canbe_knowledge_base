- **一句话定义**
	- Python API 调用是指用 Python 客户端或 HTTP 请求连接外部服务，发送请求、解析响应，并处理认证、超时、重试、异常和日志。

- **解决的问题**
	- 让 Python 程序调用模型服务、业务系统、第三方平台或内部微服务。
	- 把外部服务能力封装成可复用函数、类或客户端。
	- 在网络不稳定、服务失败和返回格式变化时保持程序可控。

- **核心流程**
	- 明确接口地址、请求方法、参数、认证方式和返回格式。
	- 构造请求体或查询参数。
	- 设置超时，发送请求。
	- 检查状态码或服务返回码。
	- 解析响应内容，转换为业务结构。
	- 对失败场景做异常处理、重试、降级或提示。

- **LLM API 调用要点**
	- Prompt、messages、model 和生成参数要集中管理，避免散落在业务代码里。
	- 对话式调用要保留必要上下文，但要控制历史长度。
	- 生成结果要做格式校验，不要默认模型输出总是符合预期。
	- 凭据和服务地址不要写死在公开页面或代码中。

- **工程要点**
	- 必须设置超时，避免请求永久阻塞。
	- 对可重试错误做有限重试，对参数错误和认证错误不要盲目重试。
	- 记录请求耗时、失败类型和关键业务上下文。
	- 封装客户端边界，让业务代码不直接依赖底层请求细节。
	- 交互式脚本要处理退出条件、空输入和异常中断。

- **面试表达**
	- Python API 调用如何做稳定性处理？ #card
		- 要设置超时、错误处理、有限重试、返回校验、日志和凭据管理。调用 LLM 时还要管理 prompt、上下文长度和输出格式。

- **常见误区**
	- 误区：API 调用就是把示例代码跑通。
		- 工程里还要处理失败、慢请求、格式变化、限流和安全。
	- 误区：模型 API 输出可以直接使用。
		- LLM 输出需要校验和兜底，尤其是分类、结构化抽取和代码生成场景。
