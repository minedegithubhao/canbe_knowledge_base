- **一句话定义**
	- Python 面向对象是用类和对象组织状态与行为的编程方式，核心关注封装、继承和多态。

- **解决的问题**
	- 把数据和操作数据的方法放在一起，降低复杂业务的组织成本。
	- 通过继承复用共同逻辑，通过多态让同一接口适配不同对象。
	- 用类封装内部状态，减少外部直接修改对象内部细节。

- **核心机制**
	- 类定义对象的属性和方法，对象是类实例化后的具体实体。
	- 封装把状态和行为收在对象内部，通过公开方法暴露稳定接口。
	- 继承让子类复用父类公共属性和方法，也可以重写父类方法。
	- 多态让调用方关注对象能做什么，而不是对象具体是什么类型。
	- Python 更偏鸭子类型：只要对象提供同名行为，就能被同一调用方式处理。

- **典型流程**
	- 先抽象出类的职责。
	- 再定义实例属性和实例方法。
	- 用构造方法初始化必要状态。
	- 对外提供清晰方法，避免到处直接改内部属性。
	- 需要复用时再考虑继承，避免为了继承而继承。

- **项目中怎么用**
	- Web 后端中，用类表达服务、客户端、配置对象和领域对象。
	- 数据任务中，用类封装复杂处理器、连接器和策略对象。
	- 面试中要能把封装、继承、多态和 Python 鸭子类型放在一起讲清楚。

- **面试表达**
	- Python 面向对象三大特性是什么？ #card
		- 封装把数据和操作收在对象内部，继承让子类复用和扩展父类能力，多态让不同对象通过同一接口表现出不同行为。Python 中多态不强依赖继承，更常体现为鸭子类型。

- **常见误区**
	- 误区：Python 多态必须依赖继承。
		- Python 更关注对象是否有需要的方法，不一定要求继承同一个父类。
	- 误区：双下划线就是绝对私有。
		- 双下划线会触发名称改写，主要减少外部误用，不是安全边界。
