	- ## 🧠 一句话说清楚（费曼）
		- **CLIP**  = [[图片编码器]]（CLS文本向量） + [[文本编码器]]（CLS图片向量）的跨模态模型。
		  id:: 69aac38a-726b-463d-90aa-0cba0ef3e8c8
			- 文本分支：文本 → Token 序列 → Text Transformer → 文本 CLS 向量（比如 768 维，一个向量，文本自身空间）→ 文本投影层 → 最终文本向量（比如 512 维，共享空间）；
			- 图片分支：图片 → ViT patch 序列 → Image Transformer → 图片 CLS 向量（比如 768 维，一个向量，视觉自身空间）→ 图片投影层 → 最终图片向量（比如 512 维，共享空间）；
		- **CLS文本向量/CLS图片向量**对齐后[[维度数]]相同且在**同一向量空间**，因此图文向量能“对话”
	- ## 💘企业开发场景
		- **[[图文检索]]（最常用）**
			- 文搜图：输入 “一只黑色的猫” → 召回相似猫图
			- 图搜图：上传一张图 → 召回相似图
			- 图搜文：上传一张图 → 召回描述这张图的文本
			- [[图文RAG流程图]]
		- **[[图文RAG]] （常用）**
			- 把图片 / 图文都转成CLS向量
			- 存入向量库，支持混合检索（文本 + 图片一起搜）
			- [[图文RAG流程图]]
		- 零样本图像分类
			- 不用标注数据，直接用文本 prompt 分类
			- 例：给一张图，prompt 是 “这是猫还是狗？”，CLIP 算相似度给出答案
		- 图文对齐校验
			- 电商：检查商品图和标题是否匹配
			- 内容审核：检查图片和文案是否一致

	- ## 🔁核心原理/流程（极简版）
		- **CLIP**本身就是 双 [[Only-Encoder]] 架构:
			- [[图片编码器]]
				- 主流： [[ViT]]
				- 作用：把图片变成向量
				- 本质： [[Only-Encoder]]  结构
			- [[文本编码器]]
				- 主流：**[[Transformer Encoder]] **
				- 作用：把文本变成向量
				- 本质：也是 [[Only-Encoder]] 结构
	- ##  📘 核心概念（官方）
		- CLIP = 能把 “文本” 和 “图片” 编码到同一个向量空间的[[双塔模型]]
		- 训练过程：
			- 给模型看海量**图文对**（比如一张猫的图 + 文字 “一只猫”）
		- 训练目标：
			- 让**匹配的图文对向量尽可能接近**
			- 让**不匹配的图文对向量尽可能远离**
		- 训练结果：
			- 图片 → CLS 向量（单个向量）
			- 文本 → CLS 向量（单个向量）
			- 两者在**同一个向量空间**，可以直接算相似度
