- **题目**
	- Python 装饰器是什么？

- **标准回答**
	- 装饰器是在不修改原函数代码的前提下，为函数增加额外行为的机制。本质上通常是一个接收函数并返回新函数的函数，常用闭包实现。
	- 一个合格的通用装饰器要能接收原函数参数，并返回原函数执行结果。带参数装饰器通常还会再包一层函数，用于接收装饰器自己的参数。

- **回答结构**
	- 先说作用：不改原函数，增强行为。
	- 再说本质：函数对象、闭包、重新绑定。
	- 再说工程用途：鉴权、日志、计时、缓存、重试、事务。
	- 最后补充参数和返回值处理。

- **项目追问**
	- 装饰器为什么适合做鉴权和日志？
		- 鉴权和日志是横切逻辑，很多函数都需要，但不应该污染每个业务函数主体。
	- 写装饰器时最容易漏什么？
		- 容易漏传参数、漏返回原函数结果，也容易丢失原函数元信息。
