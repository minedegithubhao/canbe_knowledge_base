- 黄金搭档
- | **你的策略** | **解决了什么问题** |
  | ---- | ---- | ---- |
  | **摘要向量** | 解决了**代码噪音**。你搜的是“方法”，摘要里全是“方法”，不会被 `a[len(a):]` 这种语法干扰。 |
  | **Meta 过滤** | 解决了**搜索范围**。直接把“类”那一章物理屏蔽，哪怕它的摘要写得再天花板级别也没用。 |
  | **Query 改写** | 解决了**表达差异**。把用户的“那节”变成“5.1 列表详解”，把“方法”变成“append, pop...”。 |
- 实际企业开发当中如下
	- ```text
	  1.改写用户问题同时提取结构话信息，格式如下，记得开启JSON Mode模式
	  	{
	  	  "mode": "filtered",
	  	  "metadata_filter": {"section": "5.1"},
	  	  "refined_query": "Python 列表对象内置方法, list.append, list.extend, list.pop, list.sort, list.reverse",
	  	  "reason": "用户指定了 5.1 章节，目标是找列表方法。"
	  	}
	  	1.1没有提取到section信息则：
	  		全局检索（摘要向量） + 关键字检索
	  	1.2可以提取到
	  		就走过滤检索（摘要向量 + meata过滤）  + 关键字检索
	  返回原文给到模型
	  ```
