- **题目**
	- Python 迭代器和生成器有什么区别？

- **标准回答**
	- 迭代器是按需返回下一个元素的对象，需要维护当前遍历状态。生成器是一种更简洁的迭代器写法，用函数式语法按需产出值，并自动保存局部变量和执行状态。
	- 手写类迭代器通常要维护位置、数据和取下一个元素的逻辑；生成器能省掉这些样板代码，更适合写流式处理和惰性序列。

- **回答结构**
	- 先说迭代器是协议和对象能力。
	- 再说生成器是更简洁的迭代器实现。
	- 再说惰性计算和状态保存。
	- 最后补充两者都可能被消费完，不能随意重复遍历。

- **项目追问**
	- 为什么生成器适合大文件处理？
		- 它可以按需产出数据，不需要一次性把所有内容放进内存。
	- 生成器和列表有什么区别？
		- 列表一次性保存所有元素，生成器按需产出元素，更省内存但通常只能顺序消费。

	- 不要遗漏生成器自动保存局部变量和执行状态。
