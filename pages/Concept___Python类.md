- **一句话定义**
	- Python 类是创建对象的模板，对象是类实例化后的实体，实例属性保存状态，方法定义对象行为。

- **解决的问题**
	- 把同类对象的公共行为集中定义。
	- 为复杂状态提供初始化、展示、比较、清理等统一入口。
	- 让代码从零散函数和字典，升级为职责更清晰的对象结构。

- **核心机制**
	- 实例化类会创建对象。
	- 实例方法的第一个参数通常写作 self，表示当前调用该方法的实例对象。
	- 构造方法用于初始化实例属性。
	- 字符串展示方法用于定义对象被打印或转成字符串时的表现。
	- 析构方法不适合作为关键资源释放的唯一依赖，文件、锁和连接更推荐上下文管理器。
	- 双下划线属性会触发名称改写，常用于减少外部直接访问。

- **典型流程**
	- 定义类名和职责。
	- 在构造方法中初始化必要属性。
	- 用实例方法操作实例状态。
	- 必要时定义字符串展示方法，提升调试可读性。
	- 通过公开方法控制外部如何访问或修改内部状态。

- **项目中怎么用**
	- 用类封装配置、客户端、服务对象、策略对象和数据处理器。
	- 如果一个函数需要共享很多状态，或函数参数越来越多，通常可以考虑抽成类。
	- 如果类只是薄薄包一层且没有状态，普通函数可能更简单。

- **面试表达**
	- self 是什么？ #card
		- self 指向当前实例对象。谁调用实例方法，self 就绑定到谁。它不是关键字，但按约定必须显式写在实例方法第一个参数位置。

- **常见误区**
	- 误区：Python 创建对象需要 new 关键字。
		- Python 通常直接调用类名创建实例。
	- 误区：实例外部随意添加属性是好习惯。
		- Python 允许动态添加属性，但工程中更推荐在构造方法里明确初始化，提升可读性和可维护性。
