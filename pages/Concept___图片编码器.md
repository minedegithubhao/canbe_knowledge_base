- **一句话定义**
	- 图片编码器是把图片转换成向量或视觉 token 序列的模型，用于图片理解、图文检索和多模态 RAG。

- **解决的问题**
	- 将图片从像素数据转换成模型可检索、可匹配、可理解的向量表示。
	- 支撑图片相似检索、文搜图、图文对齐和多模态 LLM 输入。

- **核心机制**
	- 图片先被切成 patch 或经过卷积/视觉骨干网络处理。
	- 每个 patch 或区域被编码成向量。
	- 模型聚合局部和全局信息，输出图片向量或视觉 token。
	- 在 CLIP 等模型中，图片向量会被投影到和文本向量对齐的空间。

- **典型流程**
	- 输入图片。
	- 切分 patch 或提取视觉特征。
	- 编码成视觉向量序列。
	- 池化或投影成全局图片向量。
	- 写入向量库或传给多模态模型。

- **项目中怎么用**
	- 图文检索中，图片编码器负责把图片入库为向量。
	- 图文 RAG 中，图片编码器通常和 OCR、caption、文本编码器一起构建多路证据。
	- 多模态 LLM 中，图片编码器负责把视觉输入转换成语言模型可消费的表示。

- **面试表达**
	- 图片编码器在图文 RAG 中做什么？ #card
		- 它把图片转换成向量或视觉 token，让系统可以按视觉语义检索图片，或把图片信息接入多模态模型。但它通常需要和 OCR、caption、metadata、文本检索配合，才能覆盖图片中的文字和细节。

- **常见误区**
	- 误区：图片编码器等于 OCR。
		- 图片编码器提取视觉语义，OCR 提取图片中文字，两者解决的问题不同。
	- 误区：图片只需要一个全局向量。
		- 全局向量适合粗粒度检索，复杂页面、表格和流程图可能需要区域级切分、OCR 或 caption。
