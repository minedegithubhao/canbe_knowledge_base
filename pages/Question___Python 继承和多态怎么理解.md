- **题目**
	- Python 继承和多态怎么理解？

- **标准回答**
	- 继承解决的是复用和扩展问题：子类可以复用父类公共能力，也可以重写父类方法。多态解决的是统一调用问题：调用方用同一接口调用不同对象，不同对象给出不同表现。
	- Python 的多态更偏鸭子类型，不一定要求对象继承同一个父类，只要对象提供调用方需要的方法，就可以被同一流程使用。

- **回答结构**
	- 先区分继承和多态的目的。
	- 再说重写和 super。
	- 再说 Python 鸭子类型。
	- 最后补充继承滥用和组合替代继承。

- **项目追问**
	- 多继承要注意什么？
		- 要关注 MRO 方法解析顺序，避免多个父类同名方法带来不可预期调用。
	- 什么时候组合比继承更好？
		- 只是复用能力而不是严格“是一种”关系时，组合通常更灵活。

	- 不要把继承和多态混成一个概念。
	- 不要忽略 Python 不强制依赖继承实现多态。
