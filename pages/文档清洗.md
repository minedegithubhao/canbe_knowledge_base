	- 文档清洗是什么？
	  collapsed:: true
		- 文档清洗就是在入库前，把原始文档里对检索和生成没帮助，甚至有干扰的内容处理掉，让后续的 **向量化、召回、重排、回答生成** 更稳定。
	- 文档清洗的目的？
	  collapsed:: true
		- 让文档更干净，但不失真；更规整，但不失去业务语义。
	- 清洗到什么程度算好？
	  collapsed:: true
		- 保留有业务价值的信息，去掉影响检索和理解的噪声，就算清洗得够好。
		- ![image.png](../assets/image_1776304182119_0.png)
	- 文档清洗的常见手段
	  collapsed:: true
		- 去噪
		  collapsed:: true
			- 目的：去掉无意义内容
			- 比如：
			  collapsed:: true
				- 页眉页脚
				- 页码
				- 目录
				- 广告语
				- 水印文本
				- 重复标题
				- 无意义空行
				- 特殊乱码字符
		- 格式标准化
		  collapsed:: true
			- 目的：把文档统一成更适合处理的格式
			- 比如：
			  collapsed:: true
				- 全角半角统一
				- 空格统一
				- 换行统一
				- 标点统一
				- 日期格式统一
				- 数字和单位格式统一
			- 例如：
			  collapsed:: true
				- ```text
				  ２０２４年 → 2024年
				  多个空格 → 一个空格
				  ```
		- 结构提取
		  collapsed:: true
			- 目的：尽量保留文档原本结构
			- 比如：
				- 一级标题
				- 二级标题
				- 段落
				- 列表
				- 表格
				- FAQ 问答对
		- 重复内容清理
		  collapsed:: true
			- 目的：去掉重复段落、重复页内容、模板化重复文本
			- 比如：
				- 每页都重复的公司声明
				- 重复出现的版权说明
				- 同一段内容被复制多次
		- 脏文本修复
		  collapsed:: true
			- 目的：处理 OCR 或 PDF 转文本时常见的问题
			- 比如：
				- 乱码
				- 错行
				- 断句错误
				- 单词被硬拆开
				- 表格列错位
		- 语义增强型清洗
		  collapsed:: true
			- 目的：有些场景会做轻度增强
			- 比如：
			  collapsed:: true
				- 给表格补标题
				- 把图片标题和说明拼回正文
				- 把 FAQ 问题和答案合并成统一格式
				- 给规则文档补上章节路径
			- 例如：
				- ```text
				  退款规则
				  24小时内全额退款
				  
				  增强成：
				  标题：退款规则
				  内容：订单支付后24小时内可申请全额退款
				  ```
