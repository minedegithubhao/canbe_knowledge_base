- **一句话定义**
	- Python 文件操作是通过 open 打开文件对象，再读取、写入、追加或复制内容的基础 IO 能力。

- **解决的问题**
	- 读取配置、文本、日志和数据文件。
	- 将程序结果写入本地文件。
	- 处理文本文件和二进制文件。
	- 在异常场景下正确释放文件资源。

- **核心机制**
	- open 返回文件对象。
	- 常用模式包括只读、写入、追加，以及二进制模式。
	- 写入模式会清空已有文件内容，追加模式会把内容写到文件末尾。
	- 文本文件应显式指定编码，减少跨环境乱码。
	- 文件对象使用完必须关闭，更推荐用 [[Concept/Python上下文管理器]] 自动关闭。

- **典型流程**
	- 明确文件路径、模式和编码。
	- 用上下文管理器打开文件。
	- 按需要使用 read、readline、readlines 或迭代文件对象读取。
	- 大文件不要一次性读入内存，优先分块或逐行处理。
	- 写入时明确是覆盖写还是追加写。

- **项目中怎么用**
	- 读取配置和样本文件时，优先显式写编码。
	- 复制图片、音频、视频等非文本文件时使用二进制模式。
	- 生产代码中要处理文件不存在、权限不足、编码错误和磁盘写入失败。

- **面试表达**
	- Python 文件读写如何避免资源泄露？ #card
		- 使用 with open 打开文件，把读取或写入逻辑放在 with 块内。代码正常结束或抛异常时，文件对象都会执行退出逻辑，自动关闭资源。

- **常见误区**
	- 误区：写入模式只是追加内容。
		- 写入模式会清空已有内容；追加模式才是在尾部追加。
	- 误区：所有文件都能当文本读。
		- 图片、音频、视频等二进制文件要使用二进制模式。
