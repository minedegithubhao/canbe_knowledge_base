- **题目**
	- Python API 调用如何做稳定性处理？

- **标准回答**
	- Python API 调用要把网络和外部服务都当成不可靠边界。请求必须设置超时，避免永久阻塞；对临时网络错误、服务繁忙等可重试错误做有限重试；对认证失败、参数错误不要盲目重试；响应要检查状态码、服务返回码和数据结构；失败时要记录日志并给出降级或明确错误。
	- 如果调用 LLM API，还要管理 prompt、上下文长度、生成参数和输出格式。模型输出不能直接假设正确，结构化任务要做校验，失败时要有重试、兜底或人工处理路径。

- **回答结构**
	- 超时。
	- 异常分类。
	- 有限重试。
	- 响应校验。
	- 日志和降级。
	- 凭据和配置管理。
	- LLM 输出校验。

- **项目追问**
	- 为什么不能无限重试？
		- 会放大故障、拖垮服务，也可能造成重复写入或成本失控。
	- LLM 分类接口为什么要校验输出？
		- 模型可能输出不符合格式的文本，如果直接入库或进入业务逻辑，会造成不可控错误。
