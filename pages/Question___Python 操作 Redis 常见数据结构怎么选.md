- **题目**
	- Python 操作 Redis 常见数据结构怎么选？

- **标准回答**
	- Redis 数据结构要按业务语义选择。单个缓存值、计数器和开关适合字符串；对象多字段适合哈希；需要顺序消费或保存最近记录可以用列表；需要去重和集合运算用集合；需要按分数排序、排行榜或延迟任务用有序集合。缓存类 key 要设置 TTL，并考虑序列化、key 命名、连接池和缓存一致性。

- **回答结构**
	- 字符串：缓存值和计数。
	- 哈希：对象字段。
	- 列表：顺序队列。
	- 集合：去重关系。
	- 有序集合：排行榜和带权排序。
	- 工程补充：TTL、连接池、序列化、一致性。

- **项目追问**
	- 为什么缓存要设置过期时间？
		- 避免脏数据和无用 key 长期堆积，也能降低内存风险。
	- Redis 操作为什么要考虑原子性？
		- 并发请求下多个操作之间可能被打断，导致计数、库存、锁或状态不一致。
