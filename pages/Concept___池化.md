- **一句话定义**
	- 池化是把一组 token 向量、chunk 向量或局部特征聚合成一个固定长度向量的操作。

- **解决的问题**
	- 文本编码器通常输出一串 token 向量，但检索、聚类和分类常需要一个整体句向量或文档向量。
	- 池化把变长序列变成固定长度表示，便于相似度计算、向量入库和下游模型使用。

- **核心机制**
	- 平均池化：对所有 token 或 chunk 向量取平均，简单稳定，常用于通用文本表示。
	- CLS 池化：取模型特殊 token 的输出向量，依赖模型是否训练过让 CLS 承载整体语义。
	- Max 池化：每个维度取最大值，突出强特征，但可能丢失整体语义。
	- 池化结果是否好，取决于模型训练方式、输入长度、噪声和任务目标。

- **典型流程**
	- 文本被分词并输入编码器。
	- 编码器输出 token 向量序列。
	- 选择平均、CLS、Max 或其他池化方式。
	- 得到句向量、段落向量或文档向量。
	- 用于检索、聚类、分类或匹配。

- **项目中怎么用**
	- 文档聚类中，可以把文档的多个 chunk 向量池化成一个文档向量，再做主题聚类。
	- 向量检索中，embedding 模型通常内部已经完成某种池化，用户直接拿到句向量。
	- 如果自己聚合长文档，要注意池化可能稀释关键信息，必要时结合标题、摘要或分层索引。

- **面试表达**
	- 平均池化、CLS 池化、Max 池化有什么区别？ #card
		- 平均池化稳定通用，把所有 token 向量取平均；CLS 池化取特殊 token 的向量，依赖模型训练方式；Max 池化突出每个维度的强特征，但可能丢失整体语义。
	- 为什么文本 embedding 需要池化？ #card
		- 因为模型通常输出一串 token 向量，而检索和聚类需要一个固定长度的整体向量，所以要用池化把序列表示压缩成句向量或文档向量。

- **常见误区**
	- 误区：CLS 池化一定最好。
		- 只有模型训练时让 CLS 学到整体语义，CLS 池化才可靠。
	- 误区：平均池化没有信息损失。
		- 平均会平滑强特征，长文档或噪声多时可能降低信噪比。
