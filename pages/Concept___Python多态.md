- **一句话定义**
	- Python 多态是让不同对象通过同一调用方式表现出不同行为的机制，常体现为鸭子类型。

- **解决的问题**
	- 让调用方依赖稳定接口，而不是依赖具体类型。
	- 让不同实现可以被同一流程处理。
	- 降低业务代码中大量类型判断的需求。

- **核心机制**
	- 传统多态常通过继承和方法重写实现。
	- Python 中更常见的是鸭子类型：只要对象有需要的方法，就可以被调用方使用。
	- 多态关注行为契约，调用方关心对象能不能做某件事。
	- 接口不清晰时，多态会变成运行时错误，所以工程中要配合测试、类型提示或抽象基类。

- **典型流程**
	- 设计统一调用入口。
	- 让不同对象提供同名方法。
	- 调用方只调用这个方法，不写大量类型分支。
	- 对关键接口补充测试或类型约束。

- **项目中怎么用**
	- 多种数据源客户端都提供读取方法，调度器只依赖读取接口。
	- 多种特征处理器都提供转换方法，流水线只关心统一转换行为。
	- 多种策略对象都提供执行方法，业务流程按配置选择策略。

- **面试表达**
	- Python 鸭子类型是什么？ #card
		- 鸭子类型的意思是，如果一个对象具备调用方需要的行为，就可以被当成对应角色使用，而不必强制继承某个父类。Python 多态更多关注行为，而不是具体类型。

- **常见误区**
	- 误区：多态必须先有父类。
		- Python 中只要对象满足行为约定，也可以形成多态效果。
	- 误区：完全不需要类型约束。
		- 大项目中仍需要测试、类型提示或抽象接口帮助维护行为契约。
