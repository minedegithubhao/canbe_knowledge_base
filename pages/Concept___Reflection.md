- **一句话定义**
	- Reflection 是让模型或 Agent 对自己的输出、计划或执行结果进行检查，并基于反馈迭代改进的模式。

- **解决的问题**
	- 模型第一次输出可能遗漏要求、格式不对或逻辑不完整。
	- Agent 执行动作后需要根据结果判断是否继续、重试或修正。
	- 对复杂任务，引入检查环节能提高稳健性。

- **典型流程**
	- 生成初稿、计划或动作。
	- 用模型、规则、测试或人工检查结果。
	- 找出错误、缺口或不满足约束的地方。
	- 生成修正方案。
	- 必要时循环多次，但要设置停止条件。

- **项目中怎么用**
	- 文档整理时，先生成页面，再按规则自检 Logseq 语法、链接和隐私。
	- 代码 Agent 可先写代码，再运行测试，根据错误修复。
	- RAG 答案可先检查是否有证据支撑，再决定输出或拒答。

- **面试表达**
	- Reflection 有什么作用和风险？ #card
		- 它能让 Agent 检查和改进自己的输出，提高复杂任务稳定性。风险是增加成本和延迟，也可能陷入无效循环，所以要有明确评估标准和停止条件。

- **常见误区**
	- 误区：反思一定能发现错误。
		- 如果检查标准模糊，模型可能只是重新包装错误。
	- 误区：反思循环越多越好。
		- 过多循环会增加成本、延迟和不确定性。
