- HNSW和IVF_FLAT如何选择
  id:: 69d70d9c-5088-4385-b470-2c7b82be26ba
	- **HNSW：小数据、速度快、精度高 → RAG / 客服首选**
	- **IVF_FLAT：大数据、省内存、精度一般 → 海量检索用**
	- | 维度 | **HNSW** | **IVF_FLAT** |
	  | ---- | ---- | ---- |
	  | **速度** | **极快（毫秒级）** | 慢（需要先查聚类） |
	  | **精度** | **最高** | 中等 |
	  | **内存占用** | 较高 | **很低** |
	  | **构建速度** | 慢 | 快 |
	  | **适合数据量** | **1 ~ 100 万** | **100 万～1 亿** |
	  | **企业 RAG 常用** | ✅ **99% 项目都用它** | ❌ 极少用 |
	  | **你的智能客服** | ✅ **最适合** | ❌ **不适合** |
- 参数
	- ```
	  params={"nlist": 128}
	  ```
	- 把数据分成多少个 “小区 / 桶”，公式：nlist ≈ 数据量开平方，100 万条 → nlist=1024
