- **一句话定义**
	- Python 装饰器是在不修改原函数代码的前提下，为函数增加额外行为的语法和函数包装机制。

- **解决的问题**
	- 把鉴权、日志、计时、缓存、重试、事务等横切逻辑从业务函数中抽离出来。
	- 在保持调用方式基本不变的情况下增强函数行为。
	- 复用通用包装逻辑，减少重复代码。

- **核心机制**
	- 装饰器本质上通常是一个接收函数并返回新函数的函数。
	- 装饰器常由闭包实现，内部函数在调用原函数前后插入额外逻辑。
	- 支持被装饰函数的参数时，包装函数通常需要接收不定长位置参数和不定长关键字参数。
	- 被装饰函数有返回值时，包装函数要返回原函数的执行结果。
	- 带参数装饰器通常是三层函数：接收装饰器参数、接收被装饰函数、执行包装逻辑。

- **典型流程**
	- 定义装饰器函数，接收原函数。
	- 定义内部包装函数，加入增强逻辑。
	- 在包装函数中调用原函数并返回结果。
	- 返回包装函数。
	- 用装饰器语法绑定到目标函数。

- **项目中怎么用**
	- Web 接口中做鉴权、限流和日志。
	- 数据任务中做重试、耗时统计和异常监控。
	- 缓存场景中包装纯函数结果，减少重复计算。

- **面试表达**
	- Python 装饰器是什么？ #card
		- 装饰器是在不改原函数代码的情况下增强函数行为的机制，本质通常是接收函数并返回新函数的闭包。回答时要补充通用装饰器需要处理参数和返回值。

- **常见误区**
	- 误区：装饰器只是语法糖。
		- 装饰器语法背后是函数重新绑定，真正关键是函数对象、闭包和包装函数。
	- 误区：包装函数不需要返回原函数结果。
		- 如果原函数有返回值，包装函数必须返回它，否则调用方会拿到空值。
