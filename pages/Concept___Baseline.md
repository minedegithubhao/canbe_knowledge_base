- **一句话定义**
	- Baseline 是项目早期用简单、可靠、可解释的方法建立的最低可用对照，用来判断后续复杂模型和优化是否真的有效。

- **解决的问题**
	- 快速跑通数据、训练、评估和上线闭环。
	- 给后续调参、特征工程和复杂模型提供比较基准。
	- 帮助发现数据泄漏、标签错误、指标定义错误等基础问题。

- **常见 baseline**
	- 业务规则
		- 用现有规则或简单阈值做对照。
	- 简单统计
		- 例如均值、中位数、历史转化率。
	- 简单模型
		- 回归任务用线性回归，分类任务用逻辑回归或线性 SVM，树模型任务可用浅层决策树或随机森林。
	- 强 baseline
		- 表格数据常用 XGBoost、LightGBM 或随机森林作为强基线。

- **项目中怎么用**
	- 先确认数据管道、标签、指标和切分方案可靠。
	- 记录 baseline 的数据版本、特征版本、参数和指标。
	- 后续每次优化都和 baseline 比较，并结合误差分析判断收益。
	- 如果复杂模型只小幅提升但成本大，未必值得上线。

- **面试表达**
	- 为什么机器学习项目要先做 baseline？ #card
		- Baseline 能快速验证数据、特征、指标和训练流程是否跑通，也给后续复杂模型提供对照。如果复杂模型不能明显超过 baseline，就说明问题可能不在模型复杂度。

- **常见误区**
	- 误区：baseline 越简单越没价值。
		- 好的 baseline 是排错工具，也是业务收益判断的参照物。
	- 误区：复杂模型一定比 baseline 好。
		- 数据少、噪声大或特征弱时，复杂模型可能只是过拟合。
