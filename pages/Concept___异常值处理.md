- **一句话定义**
	- 异常值处理是识别明显偏离正常分布的样本或特征值，并判断它是脏数据、真实极端值还是业务风险信号后再处理。

- **解决的问题**
	- 极端值拉偏均值、方差和模型参数。
	- 距离类模型被异常点影响。
	- 训练数据中存在录入错误、单位错误或系统错误。
	- 真实风险样本被误删。

- **核心机制**
	- 统计规则
		- 使用分位数、IQR、Z-score、箱线图等方法发现异常。
	- 业务规则
		- 根据业务范围判断不可能值，例如年龄为负数。
	- 模型方法
		- 使用 Isolation Forest、One-Class SVM 等方法识别异常样本。
	- 处理方式
		- 删除、截断、Winsorize、单独标记、分箱，或保留作为风险信号。

- **典型流程**
	- 做分布分析和可视化。
	- 找到极端值和不可能值。
	- 和业务确认异常含义。
	- 区分录入错误和真实极端样本。
	- 选择删除、修正、截断或保留。
	- 用验证集检查处理前后模型效果和业务指标。

- **项目中怎么用**
	- 风控、故障检测、欺诈检测中，异常值可能正是目标信号。
	- 对 KNN、KMeans、线性模型，异常值影响更明显。
	- 对树模型，异常值影响通常较小，但仍可能影响切分和业务解释。

- **面试表达**
	- 异常值应该怎么处理？ #card
		- 先判断异常值是脏数据还是真实业务信号。脏数据可以删除或修正，真实极端值可以截断、分箱、单独标记或保留。不能机械删除所有异常值。

- **常见误区**
	- 误区：异常值都要删。
		- 异常值可能是最有价值的风险信号。
	- 误区：只用统计规则就能判断异常。
		- 统计异常不一定业务异常，必须结合业务含义。
