- LLM 与 RAG 的区别
	- LLM → 精理解  → 图片 patch 向量序列 + 文本 Token 向量序列   → Transformer
	- [[RAG]] → 检索效率  → CLS图片向量 + CLS文本向量 → 分别向量库  → 结果融合
	- | 维度 | RAG（检索） | LLM（多模态生成 / 理解） |
	  | ---- | ---- | ---- |
	  | 核心目标 | 快速找相似图文 | 细粒度理解 + 生成自然语言回答 |
	  | 图文编码形式 | 单个全局向量 | 向量序列（图片 patch / 文本 Token） |
	  | 关键原因 | 效率优先，只需全局语义匹配 | 理解优先，需保留局部 / 顺序信息 |
	  | 典型输出 | 相似图文的 ID / 内容 | 自然语言回答（如 “这是张三的照片”） |
