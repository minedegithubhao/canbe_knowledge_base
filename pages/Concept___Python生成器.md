- **一句话定义**
	- Python 生成器是一种用函数语法写出的惰性迭代器，能按需产出值并自动保存执行状态。

- **解决的问题**
	- 用更少代码实现惰性序列。
	- 避免一次性构造大列表。
	- 简化手写类迭代器时对当前位置和局部状态的维护。

- **核心机制**
	- 生成器函数在产出值后会暂停。
	- 下次继续取值时，会从暂停位置继续执行。
	- 生成器会自动保存局部变量和执行状态。
	- 相比手写类迭代器，生成器不需要显式维护很多实例变量和迭代协议细节。
	- 生成器被消费后通常不能自动重新从头开始。

- **典型流程**
	- 定义一个按需产出数据的函数。
	- 每次产出一个值后暂停。
	- 调用方通过 for 循环或取下一个值逐步消费。
	- 大数据或流式场景中，按需消费而不是一次性加载。

- **项目中怎么用**
	- 分批读取大文件、大查询结果或分页接口。
	- 构造数据处理流水线，让每一步按需处理。
	- 生成大量候选数据时，用生成器降低内存占用。

- **面试表达**
	- 生成器相比普通类迭代器省掉什么？ #card
		- 生成器会自动处理迭代协议并保存局部变量和执行状态，不需要手写类来维护当前位置、数据和下一个元素逻辑，所以代码更紧凑。

- **常见误区**
	- 误区：生成器会提前计算所有结果。
		- 生成器是惰性产出，只有被消费时才继续执行。
	- 误区：生成器可以随意多次遍历。
		- 生成器通常被消费后就结束，需要重新创建才能再次遍历。
