- **一句话定义**
	- 相关性衡量回答、证据或检索结果是否真正回应用户问题；在 RAG 评估中常用于判断是否答非所问。

- **解决的问题**
	- 区分“答案看起来通顺”和“答案真的回答了问题”。
	- 在 RAG 中评估召回证据、生成答案和用户问题之间是否匹配。

- **核心机制**
	- 检索相关性关注 query 和 chunk 是否匹配。
	- 答案相关性关注 answer 是否回应 question。
	- RAGAS 等评估方法会结合语义相似度和 LLM 判断，对答案相关性进行打分。

- **典型流程**
	- 输入问题、模型答案和必要上下文。
	- 判断答案是否覆盖核心诉求。
	- 判断是否答非所问、避重就轻或冗余。
	- 给出相关性分数和失败原因。

- **项目中怎么用**
	- RAG 评估中，如果真实性高但相关性低，说明答案可能基于事实但没有回答用户问题。
	- 检索链路中，相关性分数可用于 Rerank、过滤和拒答阈值。

- **面试表达**
	- 相关性和真实性有什么区别？ #card
		- 相关性看答案是否回应问题，真实性看答案是否基于事实和证据。一个答案可能真实但没回答问题，也可能相关但事实错误。

- **常见误区**
	- 误区：答案长就相关。
		- 长答案可能包含大量冗余，甚至避开核心问题。
	- 误区：语义相似度高就一定相关。
		- 相似度只能做参考，复杂问题还需要看是否覆盖核心诉求和约束条件。
