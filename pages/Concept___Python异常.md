- **一句话定义**
	- Python 异常处理是用结构化方式表达失败路径、恢复逻辑和资源清理的机制。

- **解决的问题**
	- 捕获文件、网络、数据解析、外部服务等不稳定边界的失败。
	- 把正常流程和错误处理分开。
	- 确保资源在失败时也能释放。

- **核心机制**
	- try 放可能失败的代码。
	- except 捕获并处理指定异常。
	- else 放没有异常时才执行的逻辑。
	- finally 放无论是否异常都要执行的清理逻辑。
	- 工程代码应避免裸捕获所有异常，除非重新抛出或记录足够上下文。

- **典型流程**
	- 只包住可能失败的最小代码块。
	- 捕获具体异常类型。
	- 在 except 中记录上下文或转换为业务异常。
	- 用 finally 或上下文管理器释放资源。

- **项目中怎么用**
	- 文件读写、数据库连接、远程调用和任务调度都需要明确失败策略。
	- 批处理任务中，异常处理要区分单条数据失败和整个任务失败。
	- 服务端代码中，异常要转成可观测日志、错误码和告警。

- **面试表达**
	- Python 异常处理怎么写？ #card
		- 基本结构是 try、except、else、finally。try 放可能失败的代码，except 处理指定异常，else 表示无异常后的逻辑，finally 做一定要执行的清理。工程里要避免裸 except，并优先用上下文管理器管理资源。

- **常见误区**
	- 误区：except 越宽越安全。
		- 过宽捕获会隐藏真实 bug，让错误静默发生。
	- 误区：有 finally 就一定要手动关闭资源。
		- 文件、锁、连接等资源更推荐使用上下文管理器，减少遗漏。
