- **一句话定义**
	- 向量检索是把 query 和候选内容表示成向量，再按相似度找到最接近候选结果的检索方式。

- **解决的问题**
	- 用语义相似度召回内容，而不是只靠关键词完全匹配。
	- 支撑 RAG、推荐、图片检索、相似问题检索和语义搜索。
	- 在大量向量中快速找到 Top K 近邻。

- **核心机制**
	- 离线阶段，把文档、图片或商品转成向量并写入 [[Concept/向量库]]。
	- 在线阶段，把用户 query 转成向量。
	- 检索阶段，用余弦相似度、内积或欧氏距离找到相似向量。
	- 大规模场景通常不会全量暴力比较，而是使用 [[Concept/近似最近邻搜索]]。
	- 召回结果通常还需要 [[Concept/重排序]]、metadata 过滤或 [[Concept/混合检索]] 修正。

- **典型流程**
	- 清洗和切分原始内容。
	- 用 embedding 模型生成向量。
	- 将向量、原文和 metadata 写入向量库。
	- 查询时生成 query 向量。
	- 向量库返回相似候选。
	- 后处理、重排，再进入下游生成或业务排序。

- **项目中怎么用**
	- RAG 中用向量检索召回语义相关 chunk。
	- 推荐系统中用用户向量和物品向量做粗召回。
	- 图文知识库中可以分别对文本和图片向量建索引，再做多路融合。

- **面试表达**
	- 向量检索解决什么问题？ #card
		- 它把文本、图片或商品转成向量，用相似度计算找到语义上接近的候选。它解决语义召回问题，但对精确词、编号、实体和过滤条件不一定可靠，所以常和关键词检索、metadata 过滤、重排一起用。

- **常见误区**
	- 误区：向量检索等于向量库。
		- 向量检索是一种检索方式；向量库是存储和索引向量的基础设施。
	- 误区：向量越相似，答案一定越正确。
		- 向量相似只能说明表示空间接近，仍可能召回主题相似但证据不支持的内容。
