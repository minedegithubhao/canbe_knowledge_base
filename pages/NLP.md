- # 文本/文档聚类
- ## 企业真实使用场景（使用率从高 → 低）
- ### LATER 知识库 / 文档库 **去重、查重、清理脏数据**
  id:: 69a7ae02-7130-415d-9be1-3c4b5790b9a0
	- **最常用、最刚需、几乎所有做 RAG 的企业都要用**
	- 场景：
	  
	  企业有一堆：制度、合同、公告、FAQ、客服文档、技术手册，里面大量：重复、版本混乱、相似文档、旧文档
	- 聚类做什么：
	  
	  把**内容高度相似**的文档自动聚成一类
	  
	  → 人工保留 1 份最优的，其他删除 / 归档
	  
	  → 让 RAG 检索**不重复、不乱、不冲突**
	- 价值：
	  
	  知识库干净 = RAG 效果直接提升，这是**RAG 工程第一步**。
- ### 海量文档 **自动分类、自动打标签**（无标注数据时）
  
  **第二常用，企业内部文库 / 知识管理必备**
- 场景：
  
  突然拿到 **1000/5000/1 万篇文档**，没有标签、没有分类、不知道里面是啥，人工分类太慢、成本太高
- 聚类做什么：
  
  不用标注、不用训练，直接聚类 → 自动分成 N 堆，每一堆人工起个名字：
	- 簇 0 → 考勤制度
	- 簇 1 → 请假流程
	- 簇 2 → 报销规范
	- 簇 3 → IT 问题
- 价值：
  
  从 “一堆乱文档” → **快速建成可用分类体系**，这是**零成本构建分类体系**的唯一手段。
- ### 客服 / 用户问题 **聚类发现高频问题**
  
  **第三常用，智能客服、对话系统必做**
- 场景：
  
  积累了 10 万条历史客服聊天 / 用户提问，不知道用户**最爱问什么**，不知道该**优先做哪些 FAQ**，不知道意图标签该怎么设计
- 聚类做什么：
  
  把用户问的相似问题聚在一起，看哪一类**数量最多** → 就是高频问题
- 输出：
	- TOP 问题：登录问题、密码问题、发货问题
	- 直接用来做：FAQ、意图识别标签、路由规则
- 价值：
  
  **让 AI 系统知道用户真正关心什么**，而不是拍脑袋设计。
- ### 相似内容推荐 / 关联推荐
  
  **内容平台、文库、法规库、教育产品常用**
- 场景：
  
  用户看一篇 “考勤制度”，想自动推荐：**相似的请假、加班、打卡制度**
- 聚类做什么：
  
  把内容相似的文档聚成一类同一类内的互相推荐
- 价值：
  
  不用人工做关联，**自动形成知识网络**。
- ### 舆情 / 评论 / 反馈 **自动发现热点 & 负面集中问题**
  
  **市场、舆情、运营部门用**
- 场景：
  
  每天几千条评论、留言、投诉，人工看不过来
- 聚类做什么：
  
  把讲**同一件事**的内容聚成簇，快速发现：
	- 最近大家都在吐槽物流
- 都在问新功能
	- 都在反应某个 bug
- 价值：
  
  **自动发现业务问题，不用人工刷评论**。
- ### 标注前的 “数据分层”：降低标注成本
  
  **AI 数据团队内部常用**
- 场景：
  
  要做文本分类 / 意图识别，但数据太多，全标注太贵
- 聚类做什么：
  
  先聚类 → 每一类挑**代表性数据标注**，不用全标
- 价值：
  
  标注成本 ↓ 50%～90%
- ### 总结
  
  只要满足下面 **任意一条**，就会上聚类：
  
  1. **文档多、乱、重复，要清理（RAG 必备）**
  2. **没有标签，想自动分类**
  3. **不知道用户 / 文档里都在说什么，想挖主题**
  4. **想找相似内容、做推荐**
  5. **想降低标注成本**
  
  一句话记住：**聚类 = 无标注数据下的自动分类、自动整理、自动发现。**
