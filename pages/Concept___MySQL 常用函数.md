- **一句话定义**
	- MySQL 常用函数是 SQL 查询中处理字符串、数值、日期和条件判断的基础工具。

- **解决的问题**
	- 在查询阶段完成格式化、截取、拼接、时间计算、四舍五入和条件分类。
	- 降低业务代码对简单数据清洗和派生字段的依赖。

- **核心机制**
	- 字符串函数：`lower`、`upper`、`concat`、`substr`、`replace`、`length`。
	- 数值函数：`round`、`format`、`floor`、`ceil`、`mod`、`rand`。
	- 日期函数：`now`、`date_add`、`date_sub`、`datediff`、`date_format`。
	- 条件判断：`case when`、`if`。

- **典型流程**
	- 先判断字段类型。
	- 再选择字符串、数值、日期或条件函数。
	- 对结果命名为派生字段。
	- 如果函数影响索引使用，考虑改写或提前生成字段。

- **项目中怎么用**
	- 统计报表、评估任务、数据清洗和后台列表查询都会用到。
	- RAG 评估中，可用日期函数筛选任务周期，用条件判断生成结果标签。

- **面试表达**
	- SQL 函数什么时候不应该滥用？ #card
		- 当函数包在索引列上导致索引失效、查询变慢时要谨慎。可以考虑范围查询、生成列、离线清洗或预计算字段。

- **常见误区**
	- 误区：所有清洗都应该放在 SQL 里。
		- SQL 适合轻量派生和过滤，复杂业务清洗通常放到 ETL 或应用层更可维护。
	- 误区：`length` 和字符数一定相同。
		- `length` 更偏字节长度，中文等多字节字符和字符数可能不同。
